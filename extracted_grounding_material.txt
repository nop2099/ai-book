TOPIC 2: LEARN BY BUILDING - SPECIFIC EXAMPLE
================================================================================

From: Claude code skills exploration (2025-10-19)

USER'S DEVELOPMENT PHILOSOPHY (as synthesized by Claude):
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

"## Development Philosophy

**Accountability is the cure for hallucination.** You prioritize verification and 
proof over agent assertions.

## Core Principles

### 1. **Comprehensive Test Output**
- Human-readable test results are mandatory
- Both unit tests and integration tests must produce clear, interpretable output
- Agents must READ and INTERPRET test results, not just check exit codes

### 2. **Scripts Over Ad-Hoc Commands**
- Multi-step operations should use executable scripts
- Avoid long chains of ad-hoc bash commands
- Scripts serve as both tools and documentation

### 3. **Verification Before Success**
- Never declare success without running tests
- Never declare "fixed" without verification
- Must explicitly state what was verified and the results
- Exit code 0 is not enough - read the actual output

### 4. **Prevent Code Overwork**
- Agents tend to add duplicative functions instead of searching for existing ones
- Half-implemented features lead to regressions
- Check for existing implementations before adding new code

### 5. **Context Economy**
- Prefer minimal onboarding (~5K tokens) over full context dumps
- Re-onboarding is cheaper than context compression (5K vs 50K tokens)
- Load only what's needed, when needed"

LESSON: Building projects teaches you about building systems. The domino game 
revealed what actually works with AI agents—not theory, but empirical discovery 
through the building process.
